# meetings

Папка зарезервирована под будущую доменную логику «встреч» (выделение сущности/сервисов поверх календарных событий).

Сейчас реальная работа с карточками встреч живёт в `src/calendar/eventNoteService.ts` и связанной инфраструктуре:

- `(calendar_id, event_id)` в frontmatter как стабильная идентичность (имя файла — человекочитаемое);
- `attendees*` в frontmatter — массивы `person_id` (связи держим по id; email остаётся в теле заметки в списке участников);
- статус участия (PARTSTAT/RSVP) берём из календаря и меняем через CalDAV write-back;
- `.base` файлы для встроенного Obsidian “Bases” (табличные представления по `file.inFolder("...")`).

Эта директория остаётся как место для развития:

- связывание встреч с людьми/проектами (в т.ч. авто‑ссылки `people: [[...]]`);
- индексация/поиск сущностей поверх `metadataCache`;
- расширенная модель встречи (не только календарный `Event`).
